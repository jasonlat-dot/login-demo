# 登录页面交互动画 PRD

## 1. 概述

一个带有动画角色伙伴的登录页面，角色会对用户交互做出反应，提供有趣且引人入胜的认证体验。页面采用两栏布局，左侧为动画角色区域，右侧为登录表单。

**参考项目:** [CareerCompass](https://github.com/arsh342/careercompass)

---

## 3. 动画角色定义

### 3.1 角色规格

| 角色 | 形状 | 颜色 | 尺寸（宽 x 高） | 层级 | 眼睛类型 |
|------|------|------|-----------------|------|----------|
| 紫色 | 圆角矩形 | #6C3FF5 | 170 x 370px | 1（最后） | 眼球型（白底 + 瞳孔） |
| 黑色 | 圆角矩形 | #2D2D2D | 115 x 290px | 2 | 眼球型（白底 + 瞳孔） |
| 橙色 | 半圆形 | #FF9B6B | 230 x 190px | 3（前方） | 裸瞳孔 |
| 黄色 | 圆角矩形 | #E8D754 | 135 x 215px | 4（最前） | 裸瞳孔 + 嘴巴线条 |

### 3.2 眼睛结构

**眼球型（紫色、黑色角色）：**
- 白色圆形背景（16-18px）
- 内部深色瞳孔（6-7px）
- 支持眨眼动画（高度收缩至 2px）

**裸瞳孔型（橙色、黄色角色）：**
- 仅有深色圆形瞳孔（12px），无白色背景

**黄色角色嘴巴：**
- 水平深色线条（50 x 4px），跟随面部移动

**橙色角色悲伤嘴巴（仅登录错误时显示）：**
- 向下弯曲弧形（28 x 14px 半圆边框），默认透明

## 4. 动画行为详解

### 4.1 空闲状态 — 鼠标跟随

| 属性 | 行为 | 过渡时间 |
|------|------|----------|
| 瞳孔 | 跟随鼠标光标位置，限制最大距离（4-5px） | 0.1s ease-out |
| 眼睛容器 | 根据鼠标偏移位置（faceX: +/-15px，faceY: +/-10px） | 0.7s ease-in-out |
| 身体倾斜 | 向鼠标方向微微倾斜（最大 +/-6deg skew） | 0.7s ease-in-out |
| 黄色嘴巴 | 跟随眼睛同样的偏移量 | 0.7s ease-in-out |

**位置计算公式：**
```
faceX = clamp(-15, (鼠标X - 角色中心X) / 20, 15)
faceY = clamp(-10, (鼠标Y - 角色中心Y) / 30, 10)
bodySkew = clamp(-6, -(鼠标X - 角色中心X) / 120, 6)
```

### 4.2 随机眨眼

| 角色 | 间隔 | 眨眼时长 |
|------|------|----------|
| 紫色 | 随机 3-7 秒 | 150ms |
| 黑色 | 随机 3-7 秒（独立计时） | 150ms |

**效果：** 眼球高度从完整尺寸动画缩小至 2px，然后恢复。

### 4.3 邮箱输入聚焦（打字状态）

**触发条件：** 邮箱输入框获得焦点

| 角色 | 身体 | 眼睛 |
|------|------|------|
| 紫色 | 倾斜 -12deg + 右移 40px，高度增至 410px | 看向其他角色方向（右下方） |
| 黑色 | 倾斜 +10deg + 右移 20px | 看向左上方 |

**时序：** 角色互相对视 800ms 后恢复鼠标跟随，身体在输入框聚焦期间保持倾斜。

### 4.4 密码输入聚焦（转头回避）

**触发条件：** 密码输入框获得焦点 且 密码处于隐藏状态

| 角色 | 身体 | 眼睛 |
|------|------|------|
| 紫色 | 倾斜 -14deg + 左移 -20px，高度 410px | 看向左上方（-5px, -5px） |
| 黑色 | 倾斜 +12deg + 左移 -10px | 看向左上方（-4px, -5px） |
| 橙色 | 正常 | 看向左上方（-5px, -5px） |
| 黄色 | 正常 | 看向左上方（-5px, -5px），嘴巴左移 |

**含义：** 角色们"礼貌地转头不看"，保护用户输入密码的隐私。

### 4.5 密码可见状态（显示密码）

**触发条件：** 用户点击眼睛图标显示密码 且 密码有内容

| 角色 | 身体 | 眼睛 |
|------|------|------|
| 全部 | 恢复中立姿态（skewX 0deg） | 看向远离表单的方向（左侧） |
| 紫色 | 特殊：周期性"偷看"（每 2-5 秒偷看一次，持续 800ms） | 偷看时：瞳孔移至 (+4, +5)；正常：(-4, -4) |

### 4.6 登录失败（摇头动画）

**触发条件：** 表单校验失败 或 模拟登录请求失败

**第一阶段 — 沮丧表情（立即）：**

| 角色 | 眼睛位置 | 瞳孔方向 |
|------|----------|----------|
| 紫色 | left:30px, top:55px | 左下方（-3px, +4px） |
| 黑色 | left:15px, top:40px | 左下方（-3px, +4px） |
| 橙色 | left:60px, top:95px | 左下方（-3px, +4px） |
| 黄色 | left:35px, top:45px | 左下方（-3px, +4px），嘴巴旋转 -8deg |
| 橙色嘴巴 | 悲伤弧线显示（opacity 0 -> 1） | — |

**第二阶段 — 摇头（延迟 350ms 后）：**

所有眼睛 + 嘴巴元素执行 CSS 摇头动画：

```
@keyframes shakeHead {
  0%, 100%  translate: 0
  10%       translate: -9px
  20%       translate: +7px
  30%       translate: -6px
  ...       （振幅逐渐衰减）
  90%       translate: -0.5px
}
时长：0.8s，缓动：cubic-bezier(0.36, 0.07, 0.19, 0.97)
```

**第三阶段 — 恢复（2.5 秒后）：**

- `isLoginError` 重置为 false
- 橙色悲伤嘴巴淡出
- 移除所有摇头 class
- 角色恢复空闲状态

**连续多次触发：** 清除上一次定时器，通过移除 class + 强制回流 + 重新添加来确保动画每次都能重新播放。

## 5. 按钮悬停动画

### 登录按钮

| 状态 | 内容 | 背景 |
|------|------|------|
| 默认 | "Log In" 白色文字 | #1a1a2e（深色） |
| 悬停 | 文字右滑淡出；紫色遮罩层出现，显示 "Log In" + 箭头图标 | #5B21B6（紫色） |

### Google 按钮

| 状态 | 内容 | 背景 |
|------|------|------|
| 默认 | Google 图标 + "Log in with Google" | #f5f5f5（浅灰） |
| 悬停 | 同样的滑出效果；紫色遮罩 + 文字 + 箭头 | #5B21B6 |

**过渡时间：** 0.3s 全属性

## 6. 表单校验

### 客户端校验规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 邮箱 | 必须匹配邮箱正则 | "Please enter a valid email address." |
| 密码 | 最少 6 个字符 | "Password must be at least 6 characters." |

### 错误展示

- 标签文字变红（#dc2626）
- 输入框下划线变红
- 错误提示框出现（红色文字、浅红背景、红色边框）
- 角色执行摇头动画

### 模拟登录失败

- 按钮文字变为 "Signing in..."（按钮禁用）
- 1.5 秒后显示 "Invalid email or password" 错误
- 角色执行摇头动画

## 10. 状态机

```
空闲（鼠标跟随）
  |
  +--> 邮箱聚焦（打字状态，角色互相对视）
  |      +--> 空闲（失焦时）
  |
  +--> 密码聚焦_隐藏（角色转头回避）
  |      +--> 密码可见（切换眼睛图标 -> 角色偷看）
  |      +--> 空闲（失焦时）
  |
  +--> 登录错误（沮丧表情 + 摇头，2.5秒）
         +--> 空闲（自动恢复）
```

注意：眨眼动画在独立计时器上运行，不受其他状态影响。
